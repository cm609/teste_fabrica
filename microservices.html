Microservices

Introdução

Este projeto trata do desenvolvimento de um modelo arquitetural conceitual e tecnológico para a produção, disponibilização e execução de software com a abordagem de microsserviços, no contexto da Secretaria de Segurança Pública do Estado de Mato Grosso.

A Computação Orientada a Serviços atualmente tem suas pesquisas voltadas para um modelo de desenvolvimento denominado de microsserviços (microservices), que consiste em produzir blocos de softwares autônomos que se comunicam para entregar uma solução completa, com implantação de módulos de baixa granularidade e atemporais, em oposição a soluções de software que consistem em um único bloco monolítico de código implantável.

O desenvolvimento de software por meio do modelo de microsserviços traz consigo um considerável aumento da complexidade no desenvolvimento e implantação dos elementos de software. Contudo, tal modelo promove uma série de características, que em certos contextos de desenvolvimento e aplicação de soluções de software, confere vantagens que justificam seu uso. A abordagem de microsserviços promove o desenvolvimento de elementos de software com baixa granularidade, que promovem consequentemente a diminuição da complexidade do módulo, o aumento da capacidade de reuso e compartilhamento dos processos de negócio na e entre organizações e otimiza os procedimentos de implantação e gerenciamento dos ambiente de produção.

Considerando o contexto de órgão e secretarias públicas, como é o caso da Secretária de Segurança Pública do Estado de Mato Grosso, que possui uma série de demandas de softwares autônomos, mas que em determinados momentos precisam se comunicar uns com os outros, a abordagem de microsserviços se mostra favorável para a resolução de tais demandas. Outra demanda importante da Secretaria de Segurança Pública do Estado de Mato Grosso é o fato de que existem muitos serviços críticos que necessitam de uma alta disponibilidade das soluções de software. Uma característica importante da abordagem de microsserviços, que vai de encontro com tal demanda, é o fato que os microsserviços são unidades de software independentes e que podem permanecer em execução mesmo que outro serviço esteja indisponível.

Objetivos

O objetivo geral do projeto é a definição de uma arquitetura conceitual e tecnológica de microsserviços (microservices) por meio de um protótipo funcional que atenda as demandas de software da Secretaria de Segurança Pública do Estado de Mato Grosso. Podemos detalhar os seguintes objetivos específicos:

  • Definir um modelo arquitetural conceitual de referência para soluções orientadas a microsserviço;
  • Definir uma arquitetura tecnológica orientada microsserviços compatíveis com a produção de software por meio da linguagem de programação Java e sistemas gerenciadores de banco de dados Oracle e MongoDB;
  • Definir um modelo de geração de logs para auditorias e análises estatísticas;
  • Definir um modelo de autenticação e autorização para a arquitetura de microsserviços proposta;
  • Utilizar tecnologias com padrões abertos e de baixo custo na arquitetura de microsserviços proposta.
