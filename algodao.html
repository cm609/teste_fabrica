Rastreamento de Algodão

Introdução

Atualmente, o estado de Mato grosso é o maior produtor de algodão no Brasil e a cada safra que se passa a produção aumenta significativamente, junto deste crescimento vem a necessidade organizacional com a ajuda de sistemas de informação focados em necessidades específicas, uma das necessidades é o rastreamento da colheita do algodão. O projeto consiste em um aplicativo para plataformas móveis, buscando uma solução economicamente viável, de baixo custo e de simples operação levando em conta as limitações do campo, utilizando de tecnologias modernas como aplicativos em plataformas móveis, com o objetivo de oferecer uma alternativa aos produtores de algodão.

A proposta foi utilizar a plataforma Android para captar dados referentes ao algodão colhido no campo e enviar para a aplicação web que faz o armazenamento e processamento desses dados. Dentre os resultados obtidos, vale ressaltar a grande aplicabilidade da solução em pequenos e médios produtores da região com possibilidade de várias melhorias. Espera-se que este projeto seja uma solução viável para os produtores rurais do estado de Mato grosso, abrindo assim um vasto panorama de oportunidades e possibilidades em nossa região não só especificamente na área de tecnologia da informação.

Apresentação das Tecnologias Utilizadas

Kotlin

O Kotlin é uma linguagem de programação desenvolvida pela JetBrains, empresa que trabalha com o desenvolvimento de IDE ́s, seu projeto teve início em 2010, em 2012 seu código foi aberto sob a licença Apache e a sua versão 1.0 foi lançada em 15 de fevereiro de 2016. Foi escolhida para este projeto por ser a linguagem oficial da plataforma Android.

ZXING

O ZXING é uma biblioteca consolidada entre os desenvolvedores Android para a realizar a leitura de códigos de barras de vários padrões e QR codes, a funcionalidade de leitura de código do aplicativo foi desenvolvida com a ajuda desta biblioteca.

MongoDB

MongoDB é um banco de dados de código aberto e orientado a documentos, ou seja, não segue o paradigma relacional, os dados são armazenados no formato JSON. Foi escolhido como banco de dados do projeto pelo motivo de não ter a necessidade remodelar a sua estrutura em casos de mudanças futuras no sistema.
